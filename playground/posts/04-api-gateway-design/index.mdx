# API 网关设计与实现

API 网关是微服务架构中最关键的基础设施之一。它是所有外部流量进入系统的唯一入口。

## 为什么需要 API 网关？

当服务数量超过 10 个，你就需要一个统一的流量入口来处理：

- **路由** — 将请求转发到正确的后端服务
- **认证** — 统一的身份验证和鉴权
- **限流** — 保护后端服务不被打垮
- **监控** — 请求级别的链路追踪

## 架构设计

```typescript
interface GatewayConfig {
  routes: Route[];
  middlewares: Middleware[];
  rateLimit: RateLimitConfig;
  circuitBreaker: CircuitBreakerConfig;
}

interface Route {
  path: string;
  method: string;
  upstream: string;
  timeout: number;
  retries: number;
}
```

### 限流算法

我们采用令牌桶算法（Token Bucket），相比漏桶算法，它允许一定程度的突发流量。

### 熔断机制

基于滑动窗口的错误率统计，当错误率超过阈值时自动熔断。

## 灰度发布

通过 Header 标记和权重路由实现灰度发布。

## 总结

API 网关的设计需要在灵活性和性能之间找到平衡。
